# 클레임 검토 (CLAIM_DATA) 계산 로직

> `update_all.py` Step 8 (222~248줄)
> 원본: `2026 data RAW_Claim.xlsx` — sheet_name=2

---

## 1. 대상 행 필터

```
cols[6] (클레임상태) 가 아래 중 하나:
  - 불승인
  - 보완요청
  - 보완완료
```

---

## 2. 원본 컬럼 구조

| 인덱스 | 컬럼 | 용도 |
|--------|------|------|
| cols[5] | 지점 (F) | AS_ 제거 후 사용 |
| cols[6] | 클레임상태 (G) | 불승인/보완요청/보완완료 |
| cols[7] | 클레임번호 (H) | 금액 합산 키 |
| cols[8] | Task | Task=1만 표시 |
| cols[10] | Warranty Stage (K) | 워런티 단계 |
| cols[11] | Claim Type (L) | 클레임 유형 |
| cols[12] | Defect Code (M) | 디펙트 코드 |
| cols[13] | 날짜 (N) | 날짜 |
| cols[18] | 차대번호 (S) | 차량 식별 |
| cols[22] | 금액 (W) | 클레임 금액 |
| cols[23] | 담당자 (X) | 담당 확인자 |

---

## 3. 금액 집계 방식

```
클레임번호(H) 단위로 금액(W) 전체 합산 (Task 무관)
    ↓
Task=1인 행을 표시할 때 해당 클레임번호의 합산금액 사용
```
→ 동일 클레임번호에 Task=2,3 행이 있어도 금액은 전체 합산값으로 통일

---

## 4. 결과 데이터 구조 (CLAIM_DATA)

```
[
  클레임번호,   # [0] H
  지점,         # [1] F (AS_ 제거)
  클레임상태,   # [2] G (불승인/보완요청/보완완료)
  ClaimType,    # [3] _compute_type() 결과 — 다른 카테고리(BRANCH_DATA 등)와 동일 기준
                #     (BSI/Warranty/TC/RECALL/Goodwill/WP/LOCAL TC)
  ClaimType_원본,# [4] L열 DMS 원본값
  DefectCode,   # [5] M
  날짜,         # [6] N (앞 10자리)
  금액,         # [7] W (클레임번호 단위 합산)
  담당확인자,   # [8] X
  차대번호,     # [9] S
  Credit수신일자 # [10] 클레임현황 시트 매핑 (승인대기 기준)
]
```

> **[3] 변경 이력**: 구버전은 K열 원본값(`Warranty Standard` 등 DMS 문자열)을 사용했으나,
> 다른 카테고리와 일관성을 위해 `_compute_type(K열, M열)` 결과값으로 통일 (2026-04-06)
