# 클레임 볼륨 (WHOLESALE_DATA) 계산 로직

> `update_all.py` Step 9 (250~309줄)
> 원본: `2026 data RAW_Claim.xlsx` — sheet_name='클레임현황'

---

## 1. 원본 시트: 클레임현황

`bmwdms_update.py`가 DMS에서 지점별 5회 다운로드 후 생성하는 시트

| 컬럼 | 내용 |
|------|------|
| kc[0] | 지점 |
| kc[9] (J) | 부품금액 |
| kc[12] (M) | 청구금액 |
| kc[14] (O) | 승인부품금액 |
| kc[17] (R) | 총승인금액 |
| kc[18] (S) = 클레임확정월 | 승인대기 기준월 (YY-MM) |
| U = 보증마감기준 | 보증마감 기준월 (YY-MM) |
| T = Claim Type | BSI / Warranty / TC/RECALL 등 |
| W = 승인여부 | '승인완료' 또는 '승인대기' |
| Y = 회계마감기준 | 회계마감 기준월 (YY-MM) |

---

## 2. 집계 기준 2가지

| 기준 | 컬럼 | 설명 |
|------|------|------|
| **보증마감** | U (보증마감기준) | BMW 보증 기준 월별 집계 |
| **회계마감** | Y (회계마감기준) | 회계 기준 월별 집계 |

→ 동일 로직으로 2회 집계 후 `WHOLESALE_DATA['보증마감']`, `WHOLESALE_DATA['회계마감']`에 저장

---

## 3. 각 기준별 3가지 집계

### charge (청구)
```
해당 기준월(U 또는 Y)이 YY-MM 형식인 행
Claim Type + 기준월 기준으로:
  amount = 청구금액(M) 합산
  parts  = 부품금액(J) 합산
  count  = 행 수
```

### approve (승인)
```
동일 조건에서:
  amount = 총승인금액(R) 합산
  parts  = 승인부품금액(O) 합산
  count  = 행 수
```

### pending (승인대기)
```
승인여부(W) == '승인대기' 인 행
→ 보증마감기준(U)이 없으므로 클레임확정월(S) 기준으로 집계
  amount = 청구금액(M) 합산
  parts  = 부품금액(J) 합산
  count  = 행 수
```

## 5. 핵심 설계 포인트

- **승인대기는 보증마감기준(U)이 없음** → U/Y 기준 집계에서 누락됨
- 해결: 승인대기는 **클레임확정월(S열=kc[18])** 기준으로 별도 집계
- 이를 통해 대시보드 승인대기 건수/금액이 정확하게 표시됨

---

## 6. 대시보드 표시

- 탭: 보증마감 / 회계마감
- 컬럼: 청구금액, 승인금액, 승인대기금액 (건수 포함)
- 2025년 / 2026년 연도별 테이블 (컬럼 너비 고정)
- Claim Type별 행 구분
