# 고품 수거 비교

> `index.html` — view-qr-match
> 접근 권한: 관리자 전용 (admin-only)

---

## 1. 개요

클레임 RAW 데이터(QR_CLAIM_DATA)와 Google Sheets 스캔 데이터를 비교하여
고품 수거 완료 여부를 확인하는 분석 화면.

**목적**: 청구된 클레임 중 고품이 실제로 수거(스캔)됐는지 추적

---

## 2. 데이터 흐름

```
매일 아침 업데이트.bat 실행
  └→ update_all.py → Claim 상세_전체 시트 집계 → QR_CLAIM_DATA embed
        └→ 기준: 클레임생성일자(N열, cc[13]) 2026년 이상 + WC로 시작하는 클레임번호

고품 수거 비교 화면 열기
  └→ 날짜/지점 필터 선택 → 조회 버튼
        └→ QR_CLAIM_DATA 필터링 (클레임생성일 기준)
        └→ Google Sheets CSV fetch (실시간)
              └→ 클레임번호 기준 매칭 → 결과 렌더링
```

---

## 3. 데이터 구조

### QR_CLAIM_DATA (HTML embed)

```javascript
// [클레임번호(WC제거), DefectCode, 차대번호뒤7, 지점, 클레임생성일]
const QR_CLAIM_DATA = [
  ["248610", "6312356700", "CE31331", "전주", "2026-03-18"],
  ...
]
```

- 생성 스크립트: `update_all.py` 8-0 스텝
- 시트: `Claim 상세_전체`, header=1
- 컬럼: cc[7]=클레임번호, cc[12]=DefectCode, cc[13]=클레임생성일, cc[18]=차대번호, cc[5]=지점

### Google Sheets CSV (실시간 fetch)

```
QR_CSV_URL: https://docs.google.com/spreadsheets/d/e/2PACX-.../pub?output=csv
```

- 컬럼 인덱스: 0=스캔시간, 1=딜러번호, 2=지점, 3=클레임번호, 4=차대번호,
  5=모델, 6=디펙트코드, 7=주행거리, 8=등록일, 9=수리일, 10=부품번호, 11=부품명, 12=Part순서

## 4. 매칭 로직

```javascript
// WC 접두어 정규화
const normNo = v => String(v).replace(/^WC/i, '').trim();

// scanMap: 클레임번호 → 최초 수거일자(스캔시간 앞 10자)
const scanMap = {};
for (const s of scans) {
  const no = normNo(s[3]);
  if (no && !scanMap[no]) scanMap[no] = String(s[0]).slice(0, 10);
}

// 매칭 결과
_qmResults = claims.map(r => ({
  no: r[0], defect: r[1], vin7: r[2], branch: r[3], date: r[4],
  matched: r[0] in scanMap,
  scanDate: scanMap[r[0]] || ''
}));
```

## 6. 테이블 컬럼

| 컬럼 | 데이터 소스 |
|------|------------|
| 클레임번호 | QR_CLAIM_DATA[0] (WC 접두 추가 표시) |
| 디펙트 코드 | QR_CLAIM_DATA[1] |
| 차대번호(뒤7) | QR_CLAIM_DATA[2] |
| 지점 | QR_CLAIM_DATA[3] |
| 클레임생성일 | QR_CLAIM_DATA[4] |
| 수거 | 매칭 여부 배지 (✓ 수거완료 / ✗ 미수거) |
| 수거일자 | scanMap에서 조회한 스캔시간 앞 10자 |
| 비고 | 미수거 시 "고품 미수거" 표시 |

## 7. 주요 함수

| 함수 | 역할 |
|------|------|
| `initQRMatch()` | 뷰 진입 시 날짜 기본값(오늘) 설정 |
| `runQRMatch()` | CSV fetch + 필터링 + 매칭 + 통계 갱신 |
| `switchQMTab(tab)` | 전체/수거완료/미수거 탭 전환 |
| `renderQMTable()` | _qmResults 기준 테이블 렌더링 |

---

## 8. Apps Script

**URL**: `QR_WEBHOOK_DEFAULT` (index.html 상수)
**POST**: 스캔 데이터 Sheets에 행 추가
**GET**: 전체 데이터 2D 배열 반환 (현재 미사용 — CSV로 대체)
**CSV**: `QR_CSV_URL` 공개 게시 URL로 fetch (CORS 문제 우회)
